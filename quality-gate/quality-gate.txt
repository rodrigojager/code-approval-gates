# Historical note

This file used to describe an older `quality-check .` contract where the command always meant a full Docker scan.

The current product contract is defined by the root README and `PLANO_MELHORIAS.md`:

- `code-approval-gates run` defaults to `--scope changed`.
- `code-approval-gates run --scope full` performs a full project scan.
- `code-approval-gates run --scope paths --path <dir>` scans selected paths.
- `quality-check . --scope changed|full|paths` remains available as the low-level deterministic gate.
- `.code-approval-gates.ignore`, `.quality-gate.ignore`, and `.semantic-gate.ignore` control ignored files.
- `--json --no-interactive` is the recommended mode for AI agents and scripts.
- `--ci` is the recommended mode for GitLab/GitHub pipelines.

Do not use this file as the source of truth for current commands. Keep it only as a migration note for earlier design decisions.
